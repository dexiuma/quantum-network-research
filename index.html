What it does

One pipeline, from query to clean data

Eight specialized agents coordinate the full workflow — discovering papers, pulling PDFs, reading figures with vision models, and turning visual data back into numbers and equations.

Multi-source search

Discover relevant papers across arXiv and PubMed with LLM-ranked relevance, then download PDFs concurrently with retry logic.

Vision-based extraction

Any vision-capable Ollama model reads graphs and tables from PDFs and recovers the underlying data points — not just the pixels.

Formula fitting

Automatically fit curves (linear, polynomial, power, exponential, log, trig) to extracted data and validate them with R² / RMSE.

Synthetic data generation

Generate dense new data points from any extracted or uploaded formula via interpolation, extrapolation, and range sampling.

Vector database & search

Everything is embedded into ChromaDB for natural-language semantic search across papers, graphs, data points, and formulas.

Built-in benchmarking

Preset and custom benchmark suites score search relevance, latency, and extraction potential — with persistent history and charts.

Upload your own

Drag-and-drop PDFs, CSV/JSON datasets, or formulas (as text or files) straight into the pipeline through the web UI.

External dataset discovery

Surface related datasets on GitHub, Zenodo, Figshare, HuggingFace, and PapersWithCode for any topic you mine.

Export everything

One-click export to JSON / CSV / ZIP, bundled with AI-generated analysis reports for downstream use.

Any model, multi-GPU

Works with every model in the Ollama library — pull, swap, and assign any model to each agent, distributed across GPUs, all from the web interface.

Why it's different

Private by design, complete by default

Most extraction tools stop at "here's the figure." PaperMiner closes the loop — turning figures back into data, data into formulas, and formulas into new data — without ever calling a cloud API.

1

Runs on any local model

Powered by Ollama on your own hardware — use any model in its library, from 1B to 70B+. No API keys, no usage costs, no documents ever leaving your machine — ideal for sensitive or unpublished research.

2

Closes the figure → data → formula loop

It doesn't just crop charts. It recovers numeric data points, fits validated formulas, and can regenerate denser datasets from those formulas.

3

True multi-agent orchestration

Eight cooperating agents each own a stage of the pipeline, so you can swap models, sources, and steps independently.

4

Measures its own quality

A built-in benchmarking suite quantifies search relevance and extraction potential, with persistent history so you can track improvements.

5

Clear data provenance

Extracted, interpolated, and user-uploaded data are labeled separately in the database, so you always know where each number came from.

Quick start

Up and running in minutes

Clone, install, and point it at your favorite local models. PaperMiner ships with an installer for both backend and frontend.

1

Install prerequisites

Python 3.9+, Node 18+, Tesseract OCR, and Ollama running locally.

2

Clone & install

Run ./install.sh to set up the Python venv and the React app in one step.

3

Pull any models

Any text model for search and any vision-capable model for figure extraction — anything in the Ollama library works.

4

Open the UI

Visit localhost:3000, pick a topic, and run the full pipeline.

bash — PaperMiner
# Clone & install (backend + frontend)
$ git clone https://github.com/dexiuma/PaperMiner.git
$ cd PaperMiner && ./install.sh

# Pull a text + vision model via Ollama
$ ollama pull llama2:7b
$ ollama pull llava:13b

# Start backend
$ source venv/bin/activate && python -m backend.main

# Start frontend (new terminal)
$ npm run dev
→ open http://localhost:3000
